# Onboarding Page Jargon Scan

This document lists every user-facing word or phrase from `site/onboarding.html` that a non-technical startup founder might not understand, along with plain-English replacement suggestions.

## Findings

- **Sims** (used repeatedly throughout the page) → AI customer personas
- **churn** (Step 0: "before they churn") → leave your product / stop using it
- **widget** (Step 2: "Embed the bug-report widget", "the widget puts...") → small tool that lives on your website
- **embed** (Step 2 tile title: "Embed the bug-report widget") → add to your site
- **snippet** (button label: "Copy snippet") → code piece / paste this block of code
- **script tag** (widget description: "One script tag on your site") → HTML code line / a piece of code you paste into your site
- **extension** (Step 2 tile title: "Connect the browser extension", mentioned in several places) → Chrome add-on for Klavity
- **monitored URLs** (Step 2: "Which pages should your Sims watch?", success message: "Extension connected — your Sims can now go live on monitored URLs") → pages we'll watch
- **URL pattern** (Step 2 error message: "Enter a URL pattern (e.g. app.acme.com/*)") → web address format
- **theme.liquid** (Shopify install guide) → the main theme file in Shopify
- **Code Injection** (Squarespace install guide) → Squarespace's "extra code" section (already fairly clear, but could be simplified to "add extra code")
- **Body — End of tag** (Wix placement option) → just before your website closes / at the bottom of each page
- **footer.php** (WordPress note in Other platform guide) → the footer file in WordPress themes
- **transcript** (Step 3: "Upload a transcript") → written record of a call

## Total jargon items found: 14
