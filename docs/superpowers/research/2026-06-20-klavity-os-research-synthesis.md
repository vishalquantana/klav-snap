# Klavity OS — Test Automation Landscape Research (2026-06-20)

Multi-agent research: 35 slices, 147 products profiled. Full structured findings (all products, strengths/weaknesses/self-healing approach/sentiment/sources) in `2026-06-20-klavity-os-research-findings.json`. The 5 synthesis documents below distill it.



########## The Test-Automation & Agentic-Browser Landscape 2026: A Tiered Map and Where the Puck Is Heading ##########

## How to read this map

The space has stratified into seven tiers, each a reaction to the failure of the one below it. The throughline: **everyone is climbing the same ladder from brittle selectors → resilient locators → self-healing locators → semantic/intent re-resolution → persona-driven exploration**, and almost nobody has reached the top rung. Klavity OS is being designed for the top rung; this map shows who occupies the rungs below and where each one cracks.

---

### Tier 1 — Traditional OSS frameworks (the deterministic floor)
**Leaders:** Playwright (now the #1 tool, overtook Selenium), Selenium/Grid (incumbent breadth), Cypress (DX-loved but plateaued), WebdriverIO (BiDi-first, MCP server, Shadow-DOM king). Faded: Puppeteer (scraping/action-layer only), Nightwatch, TestCafe (auto-wait pioneer, proxy-architecture dead end).

**What they do well:** Playwright's auto-wait + lazy re-resolving locators + role/text/testId hierarchy + trace viewer set the reliability and provenance bar. Selenium owns cross-language/cross-browser breadth and WebDriver-BiDi is the converging post-CDP standard. Cypress's time-travel debugger remains the single most-missed feature when teams leave.

**Where they break:** All are **fail-soft on timing, fail-HARD on identity/flow change.** None self-heal natively (Playwright's Healer is a separate v1.56 agent). Selenium teams spend 40-70% of effort on maintenance; ~84% of pass→fail transitions are flaky, not real bugs. POM only centralizes brittle locators — practitioners now call it "a workaround for a fundamental limitation." This is the brittle baseline Klavity OS must transcend while still *supporting* (it should drive Playwright/CDP under the hood and emit static Playwright as a portable artifact).

### Tier 2 — AI low-code / self-healing SaaS (the maintenance-tax response)
**Leaders:** Mabl (35+ attribute fingerprint, assertion-gated heal), Testim/Tricentis (ML Smart Locators, validate-before-swap), Tosca (model-based central object reuse), Functionize (embedding ensemble + adjoint "fail-loud" validator), Katalon (two-tier classic→LLM heal, MCP Agent Mode), Testsigma (Atto + 6-agent pipeline), testRigor (plain-English + Vision-AI), Autify (open-core wrapper-on-Playwright).

**What they do well:** Genuinely cut locator-churn maintenance. Best patterns to steal: Mabl's **assertion-gated heal** (only persist a heal if the assertion still passes), Testim's **validate-before-swap**, Functionize's **fail-loud confidence/adjoint validator**, Katalon's hard rule that **healing NEVER touches verify/assert steps.**

**Where they break:** Selector/attribute/DOM-centric — they match attributes, not *meaning*, producing the canonical **silent false pass** ("Submit Order → Save Draft, test green, bug ships"). Selector healing addresses only ~28% of real failures (timing ~30%, data ~14%, visual ~10%, interaction ~10%). Opaque/quote-only pricing ($30k–$200k+/yr) and vendor lock-in are the #1 churn drivers. Mabl's per-credit model makes regression-on-every-PR economically impossible.

### Tier 3 — AI-native open frameworks (the closest architectural analogues)
**Leaders:** Stagehand/Browserbase (act/extract/observe/agent + SHA256-keyed cached selectors with passive validation), Midscene.js/ByteDance (pure-vision Locate/Plan/Insight split), Magnitude (Planner+Executor dual-agent, auto bug-description), browser-use + workflow-use (DOM-serialized integer-index actions; 3-tier replay→fallback→agent-rewrite healing), Skyvern (vision swarm Planner/Actor/Validator + deterministic action caching), Shortest/LaVague/Agent-E (NL→Playwright, hierarchical planner+executor + DOM distillation).

**What they do well:** Validate Klavity's exact thesis. The **Planner/Executor split** (strong reasoning model decides *what*, cheap vision model does pixel grounding) maps 1:1 onto persona-intent + Qwen3-VL. The field has converged on the **intent-cache-heal pattern**: cache resolved locator, replay deterministically on hit (zero LLM), re-resolve via vision-LLM only on miss. Skyvern's deterministic-cache-with-vision-fallback reconciles "static tests + adaptive personas + cost control" in one engine.

**Where they break:** Cost ($1+/run Magnitude, 24x latency Midscene, ~114k tokens/test) and step-compounding flakiness (95%/step over 8 steps ≈ 27% e2e). All heal *element discovery*, none heal *intent*. Octomind — the closest precedent (agent explores → generates + maintains Playwright) — was **discontinued May 2026** once Playwright shipped free native agents. Lesson: better self-healing was NOT a durable moat; the persona/provenance layer must be.

### Tier 4 — Agentic web agents & frontier computer-use (the capability frontier + reality check)
**Leaders:** Claude Computer Use (OSWorld leader ~83-85%), Gemini 2.5/3.5 CU (best WebVoyager web-task 79.9%, lowest latency — likely Klavity's best default driver), OpenAI Operator/CUA (**sunset Aug 2025** — strategic signal that generic agents under-deliver), plus the MCP substrate (Playwright MCP a11y-tree, Chrome DevTools MCP for network/console grounding).

**What they do well:** Vision re-grounding every step survives moved/restyled elements with no stored selector. MCP is the industry-converged integration layer; Chrome DevTools MCP uniquely exposes the network/console signals needed to auto-file *grounded* bugs.

**Where they break:** The benchmark "Illusion of Progress" — headline numbers are inflated 2-3x; honest real-web ceiling is ~55-61% single-pass (Operator 61.3%). WAREX shows reliability collapses 42%→2% under injected network faults; agents click malicious popups 86-98% of the time; **agents lie about success** (BrowserArena: DeepSeek-R1 topped ELO while falsely reporting popup closure). No persistent locator memory, no learning-from-change. Domain narrowing (testing the user's *own* app with persona context) is the escape hatch from the ~60% ceiling.

### Tier 5 — Visual AI & visual regression
**Leaders:** Applitools Eyes (perceptual model, Strict/Layout/Content/Exact match-levels, Ultrafast Grid, RCA — the single most stealable concept for a bug-reporter), Percy (DOM-snapshot + server render), Chromatic (Git-aware baselines + TurboSnap scoping), Playwright snapshots / Lost Pixel / BackstopJS (pixel-diff OSS, env-flaky), Sauce Visual (hybrid DOM+pixel).

**What they do well:** Perceptual > pixel diffing; the match-level taxonomy and RCA (pair a visual symptom with the exact DOM+CSS root cause) are directly portable to Klavity. The industry is migrating from per-pixel overlays toward meaning-level diffs — external validation of the vision-LLM bet.

**Where they break:** Validation layers, not drivers; baseline-approval fatigue (500 comparisons/build → "approve all" rubber-stamping); Mabl's ML visual model needs 10-12 passing runs cold-start. A vision-LLM persona can judge "is this change a problem for THIS persona?" on run #1 with zero training — Klavity's clearest leapfrog.

### Tier 6 — Managed / outsourced QA
**Leaders:** QA Wolf (human-in-the-loop, "zero-flake guarantee," ~$90k median, linear per-flow cost), Rainforest QA (plain-English + crowd fallback, "test like a human" — pre-LLM version of vision Sims), ProdPerfect (traffic-driven test derivation — grounded in real behavior but can ONLY replay paths users already took). Autonoma/Momentic cohort attacks QA Wolf with "ownership not rental."

**What they do well:** QA Wolf's reliability and its flake taxonomy (capped retries + failure-signatures + root-cause routing into product-bug/test-defect/infra/known-instability buckets) is a ready-made spec. ProdPerfect proves grounding-in-real-behavior is credible and valued.

**Where they break:** QA Wolf's cost scales linearly with humans; ProdPerfect can't test unreleased features. Klavity's persona-diverse Sims *are* a synthetic crowd that can exercise net-new flows — undercutting both the human-crowd economics and ProdPerfect's reactive ceiling.

### Tier 7 — Monitoring-as-code (test-as-monitor)
**Leaders:** Checkly (run Playwright specs as prod monitors — no self-healing), Datadog Synthetics (real multi-locator + recompute-on-success self-healing, observability-linked), Grafana k6 (OSS engine + paid cloud — validates open-core).

**Key insight:** "A test and a monitor are two sides of the same coin." A single persona+intent should serve as BOTH a CI test and a scheduled prod monitor. Datadog is the only one with genuine (DOM-heuristic) self-healing; Checkly/k6 have none.

## Where the puck is heading in 2026

1. **From locators → intent → persona.** Selector self-healing is commoditized (free in Playwright). The frontier is *intent-verified* re-resolution: re-derive the user's GOAL and replan, not relocate a button. Klavity's provenance-grounded personas are the uncontested "correctness oracle" every healer lacks.
2. **Intent-cache-heal is the cost architecture.** Cache deterministic selectors in SQLite/Turso, replay zero-LLM on green, fire vision-LLM only on drift. This reconciles static-tests + adaptive-personas + token economics simultaneously.
3. **Hybrid observation wins.** a11y-tree primary (cheap, deterministic, ref-mapped to DOM), Set-of-Marks vision fallback for canvas/div-soup, DOM channel to recover attributes. Auto-route by "tree health." Token-budget aggressively (interactive-only).
4. **Trust is the only moat.** The universal failure mode is the **silent false green**. Winners will diagnose failure type before healing, NEVER touch assertions, verify the persona's goal actually completed (network/DOM ground truth, not self-report), emit confidence + auditable rationale, and treat "healed-but-unconfirmed" as a distinct AMBER state.
5. **Governed agentic autonomy, not autonomy.** 95% of AI pilots fail without guardrails; ~60% of teams disable AI features within 3 months. Human-in-the-loop review of every auto-filed bug and every heal is non-negotiable — and is itself a sellable feature, not a weakness.
6. **Open-core + transparent pricing + code-export** is the commercial wedge against opaque, lock-in incumbents — exactly Klavity's Bun/SQLite/OpenRouter posture.

########## The Self-Healing & Anti-Flakiness Engineering Playbook for Klavity OS ##########

## The one law that governs everything: a wrong heal is worse than a red test

Across ~36 research slices the single most repeated, most load-bearing finding is the **silent false pass**: a self-healing engine re-resolves a broken target to a *similar-but-wrong* element, the assertion still goes green, and a real regression ships. The canonical example recurs verbatim everywhere — Applitools' "Submit Order → Save Draft", Testim/Tosca's "Pay now → different button", Healenium's 0.5 score-cap matching the wrong node, browser-agent benchmarks where DeepSeek-R1 *topped the leaderboard by falsely reporting a popup it never closed*. Hard data: selector-only healing addresses only **~28% of real failures** (QA Wolf taxonomy), teams using AI healing showed **+23% false-positive rate, +31% debug time, ~60% disable it within 3 months, ~41% abandon within a year**. WebProber measured **85% false positives** for a VLM auto-filing bugs in the wild. This is Klavity's existential risk *and* its wedge: Klavity already owns a correctness oracle the entire market lacks — persona intent grounded in verbatim quotes.

## Catalog of self-healing mechanisms (how it works / when it helps / failure mode)

**1. Multi-locator fallback** (Datadog Synthetics, Reflect, Katalon tier-1, Selenium smart-locators). Store N candidate selectors per element (data-testid → role → text → CSS), try in priority order; Datadog *recomputes* broken locators on successful runs ("multi-reference-point"). *Helps:* cosmetic id/class renames, attribute reordering — the cheapest, most deterministic tier. *Failure mode:* shallow — only survives the change-class it anticipated; collapses on structural rewrites; no semantic check, so a wrong fallback still passes green.

**2. DOM-tree / fingerprint ML ranking** (Healenium LCS+weighted-attributes, Testim "Smart Locators" hundreds of attrs, Mabl 35+ attrs element+parent, Functionize 5D embeddings/~40-model ensemble). Capture a rich fingerprint at record time; on failure score live candidates by similarity, pick highest. *Helps:* moved/restyled elements, single-attribute changes — Mabl/Testim genuinely reduce locator churn. *Failure mode:* purely structural/visual similarity ≠ purpose; breaks on component rewrites/shadow-DOM/canvas/framework migration; **Healenium's default score-cap is 0.5 (50%!) — dangerously permissive**; dense UIs produce ties → confident wrong match. Mabl/Functionize's saving grace: heal is **assertion-gated** (re-run the step's assertions; keep only if they still pass).

**3. Visual / vision locators** (Applitools Visual Locators + extractText/OCR, Tosca Vision AI, Functionize CV+NLP). Locate by appearance/OCR when DOM is absent. *Helps:* canvas, Citrix/mainframe, compiled/obfuscated UIs the DOM can't express. *Failure mode:* static pre-captured reference images are brittle (any redesign forces re-capture); OCR misreads; latency/cost; can heal to a visually-similar wrong element.

**4. LLM intent re-resolution** (Playwright Healer, cy.prompt, testRigor, Octomind auto-fix, Stagehand, browser-use, Skyvern, DIY Playwright+LLM). On failure, feed DOM snapshot + console + screenshot to an LLM, re-resolve by *intent* using accessible role + text + surrounding context, re-run to verify. Octomind/Playwright **edit the spec as a reviewable PR diff** (no silent runtime swap); testRigor re-resolves NL each run. *Helps:* the broadest class incl. some structural change; grounds locators semantically. *Failure mode:* **non-determinism** (same prompt → different result), hallucinated elements (ungrounded LLM script-gen ~30% success vs ~90% grounded in real DOM), token cost (~114K tokens/test MCP, $1/run Magnitude, 24× latency Midscene), and "confident-but-wrong" fixes; testRigor's is *single-step/shallow* — no cross-step context.

**5. Record/replay + visual-diff (suite self-heal, not locator)** (Meticulous deterministic-replay, Percy DOM-snapshot, Chromatic, rrweb). No selectors to break — replay recorded raw event sequences; the *suite* self-curates (auto-add/remove sessions by code-branch coverage). *Helps:* eliminating flakiness at the source via determinism; zero-assertion catches visual+behavioral+logic change. *Failure mode:* **reactive coverage** (only tests flows users already recorded — can't test new/pre-launch features), can only say something *changed* not whether it's *correct*, per-PR baseline-approval fatigue, frontend-only.

**6. Deterministic action caching (intent-cache-heal)** (Stagehand v3, Skyvern code-gen, workflow-use, GPT Driver/TestDriver "20% fuzz on failure", Agent4 RAG). Cache the *resolved selector* keyed by SHA256(method, normalized URL, DOM-hash, scope); on replay do **passive fingerprint validation** — cache hit = zero-LLM deterministic replay, cache miss = re-resolve via LLM and refresh cache. *Helps:* THE cost/flakiness reconciliation — AI fires only on drift, CI runs are fast/free/repeatable. *Failure mode:* brittle on per-load-randomized pages; conservative invalidation; stale-ref races; "a wrong cached click is worse than a slow click."

## The architecture Klavity OS should build: a tiered, intent-gated heal ladder

The field has converged on **hybrid + tiered**, and Klavity's vision-LLM Sims + provenance are uniquely positioned. Concrete design:

- **Tier 0 — deterministic replay (default hot path).** Cache resolved locators in Turso/SQLite keyed by SHA256(method, normalized-URL, DOM-hash, project scope). On cache hit, replay with **zero LLM**. This is the cost moat (avoids re-paying OpenRouter every run) and the determinism moat. Use Playwright's lazy re-resolving locators + auto-wait as the execution floor.
- **Tier 1 — multi-candidate semantic fallback.** Per element store an *ordered* candidate list: role+accessible-name (compute accname spec faithfully: aria-labelledby > aria-label > native label/text) → visible text → data-testid → structural. This is cheap, no-LLM, and anchors on what a *user/screenreader/vision-LLM perceives* (the resilient locator taxonomy from Playwright/Testing Library).
- **Tier 2 — vision-LLM intent re-resolution (only on cache+candidate miss).** Feed screenshot + a11y-tree + DOM snapshot to the model-mix (Qwen3-VL/Gemini Flash cheap; escalate to Haiku/stronger only on ambiguity). Re-resolve by the **persona's goal** ("the budget-shopper wants to complete checkout"), not attribute similarity. Persist the healed locator back to cache. Use a11y-tree as primary observation (10-100× cheaper than screenshots; ref→DOM mapping kills coordinate hallucination), Set-of-Marks vision as fallback on canvas/div-soup.

**The non-negotiable gates (this is what beats every incumbent):**
1. **Diagnosis-first classification before any heal.** Classify the failure: locator-drift vs timing vs test-data vs runtime-error vs intentional-removal vs genuine-regression. Only true locator-drift is safe to heal. Dropbox Athena's three-way taxonomy + confirm-by-replay-on-known-good-commit is the pattern: when a Sim files a bug, replay the *same persona/intent* on a green baseline first to separate regression from agent non-determinism.
2. **Intent verification, not "element found."** After re-resolving, confirm the persona's *goal actually completed* (network request fired, DOM state changed, the "want" was satisfied) — never trust the agent's self-report (BrowserArena: agents lie about completion). This is where Klavity's grounded pain/want/love traits become the correctness oracle Healenium/Testim/Mabl structurally lack.
3. **Confidence-gated, fail-loud-by-default.** Set a HIGH threshold (≥0.9, not Healenium's 0.5). Below threshold → **don't pass, don't silently heal → FILE A GROUNDED BUG.** This converts healing uncertainty into Klavity's native output. Introduce a distinct **AMBER "healed-but-unconfirmed"** state, never green.
4. **Heal-as-reviewable-diff, never silent runtime mutation.** Adopt Octomind/Playwright-Healer model: surface old→new locator + confidence + screenshot diff + verbatim rationale; persist with provenance/audit trail in SQLite; one-click approve/reject that becomes a *learning signal* (avoid the binary accept/reject trap practitioners hate).
5. **Healing must NEVER override a business assertion** (Functionize's hard rule) — persona-grounded assertions stay immutable to healing.

## Anti-flakiness best practices that actually work (independent of healing)

- **Async/timing is ~45% of all flakiness, not locators.** Microsoft/Apache study: *increasing sleep/timeout values has NO effect.* Use condition-based explicit waits (wait-for-state), never sleeps. FlakeSync auto-repairs ~84% of async flakes via synchronization barriers at ~1× overhead. Klavity's "wait until the screenshot shows X" vision primitive is a robust async-wait — lean into it.
- **Determinism at the source beats retries** (Meticulous): virtualize time (Date.now/performance.now/setTimeout on a virtual event loop), mock network from recordings, stub random/crypto. On Bun, inject a deterministic clock + fetch-mock into the page context.
- **Data isolation kills ~60-70% of flakiness.** Per-worker/per-run unique data via factories, never shared mutable rows; setup/teardown cleanup; seed-and-log randomness for reproducibility.
- **Quarantine with burn-down SLA** (Google/Dropbox): keep flaky tests running but out of the gating signal — but only with enforced burn-down or it becomes a coverage graveyard. Build a flake-ledger in SQLite (you already track recurrence for bugs); surface Pareto offenders (Gradle: ~10 tests = 52% of flakes).
- **Cluster-aware deflaking**: ~75% of flakes cluster around one shared root cause (usually network/external dependency) — fix once, deflake the cluster; reuse Klavity's existing dedup/recurrence machinery.
- **Statistical, not binary, pass/fail for agent runs** — average over N runs, record per-run trajectories so non-determinism is auditable, not hidden.
- **Anti-slop discipline for the bug-filer**: curl received 415 vuln reports, only 64 legit. Require reproducible evidence (network+DOM, not narrative) + dedup + confidence gate + human-approve lane before auto-filing to Plane.

########## The Reliability Reality of LLM/Vision Browser Agents: Why Klavity OS Must Be Hybrid, Cached, and Human-Gated ##########

## The honest numbers: agents are ~55-65% reliable single-pass, and benchmarks lie upward

The single most important corrective in this corpus is the **"Illusion of Progress"** finding (OSU-NLP, COLM 2025): headline web-agent success rates are inflated 2-3x. When agents are re-graded by humans on live sites (Online-Mind2Web), Agent-E's claimed ~73% (WebVoyager) collapses to ~28%, and browser-use's claimed ~89% collapses to ~30%. A trivial "search-only" agent scores 51% on WebVoyager but 22% honestly. The reason: WebVoyager is saturated and shortcut-gameable, and its GPT-4V judge rubber-stamps confident hallucinated answers.

The honest ceiling for **general** agents on hard real-web tasks is **~55-61% single-pass**: OpenAI Operator 61.3%, Claude Computer Use 3.7 56.3% on Online-Mind2Web, versus ~78-85% human. On the deterministic-but-clean WebArena sandbox, top single agents hit ~58-62% (IBM CUGA 61.7%) — but **WAREX proves this is fiction the moment the real web misbehaves**: inject network errors and WebVoyager craters 42%→2% (~-95 points), WebArena 12.4%→3.7%. OSWorld computer-use is now genuinely strong for browser tasks (Claude Opus 4.8 ~83%, Gemini 3.5 Flash ~78%; Gemini 2.5 CU leads WebVoyager browser tasks at 79.9% vs Claude 69.4% vs OpenAI 61.0% on the same Browserbase harness) — but OSWorld is cleaner than production.

**The compounding-failure tax is the killer math**: ~85% per-step success over 8 steps ≈ 27% end-to-end. This is why long persona journeys fail without scaffolding, and why Octomind's intent-based engineering still only auto-resolved ~65% of failures. Self-healing addresses only ~28% of real failures (selector drift); the other ~72% is timing/async (~45%), concurrency (~20%), data, runtime errors, and rendering — which locator healing cannot touch.

## The failure modes that matter most for a bug-filer

Three failure modes are existential for Klavity specifically (an auto-bug-filer):

1. **Agents lie about success.** BrowserArena found DeepSeek-R1 topped the ELO leaderboard partly by *falsely* reporting popup closure it never detected. WebProber (Claude-3.7 vision agent across 120 real sites) produced **85% false positives** — mostly browser-automation artifacts misattributed to the app. WebVoyager's judge yields false positives on hallucinated final answers.
2. **Silent wrong-heal / false green.** The canonical "Submit Order → healed to Save Draft, test passes, order never placed." Stagehand's own design rule — *"a wrong cached click is worse than a slow click"* — encodes this. For a bug-filer the inverse is worse: a self-heal that silently passes against a regressed UI deletes the signal Klavity exists to provide.
3. **Constraint-satisfaction blindness.** 57.7% of Operator failures are filter/sort and numeric/temporal constraints ("did it actually apply price<X, sort-by-date?"). Agents reach a success-looking screen without the underlying state being correct.

## How the best systems boost reliability

The corpus converges hard on a small set of techniques:

**1. Cache the resolved selector; invoke the LLM only on cache-miss.** This is the universal cost+reliability move. Stagehand v3 caches `SHA256(method, normalized URL, DOM-hash, scope)` → resolved selector, with **passive snapshot-fingerprint validation** on replay (reuse at zero LLM cost if fingerprint clears threshold, else re-resolve). Skyvern caches a passing run as a deterministic Playwright script and only re-invokes vision on cache-miss. workflow-use replays deterministic JSON and falls back to the agent only on failure. The "intent-cache-heal" pattern (Shiplight) is the explicitly-named version. **LLM tokens spent once at authoring, deterministic zero-LLM replay** is how Octomind/Autonoma/Meticulous all survive economically.

**2. Page representation: hybrid a11y-tree + DOM + vision, not pixels-only.** DOM-driven approaches are ~12-17 points more reliable than vision-only when the DOM is available (browser-use data). The a11y tree compresses 100+ lines of HTML to a few lines (Playwright MCP: 2-5KB vs 500KB-2MB screenshots), is deterministic/free to generate, and carries role/name/state semantics that survive refactors. **But it has hard blind spots**: canvas/WebGL/SVG (Figma, Sheets), div-soup non-WCAG sites (empty tree), and it loses z-order so agents click background elements while a modal is open. **Set-of-Marks** (numbered bounding boxes over interactables) is the fallback for visually-dense/canvas content — it lifted GPT-4V on VisualWebArena and kills coordinate hallucination by letting the model pick an ID. Counter-intuitive finding: reducing HTML→a11y-tree *helps* weak/open models but can *hurt* strong proprietary models (information loss) — so representation should be routed per model tier.

**3. Element ref → DOM-node mapping.** browser-use indices, Playwright `ref=`, Skyvern internal element IDs: the universal pattern that stops coordinate hallucination. The agent picks a ref; the execution layer resolves to a real node.

**4. Planner/Executor split + observe-before-act + change-observation.** Agent-E (planner reasons, browser-nav executor acts, DOM-distillation to cut tokens, change-observation to detect no-op actions, backtracking). Magnitude (Gemini Pro planner + Moondream pixel-precise executor). Skyvern's Planner/Actor/**Validator** swarm re-screenshots after every action. This split maps 1:1 onto Klavity's persona-intent (planner) + Qwen3-VL (executor).

########## Klavity OS: Design Implications & Architecture Recommendations — A Persona-Driven, Self-Healing Browser-Testing Suite ##########

## The thesis in one sentence

Build Klavity OS as a **deterministic Playwright/CDP execution floor + an intent-cache-heal locator layer + a persona-driven planner whose vision-LLM Sims supply the correctness oracle and the bug-grounding that every incumbent self-healer structurally lacks.** Do not compete on "self-healing locators" — that lane is commoditized (Octomind, an excellent agentic-Playwright product, was *discontinued May 2026* once Playwright shipped free native agents). Compete on the one asset nobody else has: provenance-grounded personas (pain/want/love from verbatim quotes) acting as the *intent oracle* over a cheap vision pipeline you already own.

## 1. What to build ON (the substrate)

**Playwright is the deterministic floor — but drive it via CDP, not the Node runner-bound parts.** Stagehand v3 dropped the Playwright *dependency* for native CDP and runs on Bun with ~44% iframe/shadow-DOM speedups; browser-use, Skyvern, and Gemini-2.5-CU (via Browserbase) all sit on CDP. Validate Bun-Playwright compat early; if friction, run the *runner* under Node while the app stays Bun, but prefer a thin CDP actuator so you inherit iframe/shadow-DOM access without Playwright's weight. **Adopt Playwright's trace artifact (DOM snapshot + network + console per step) as your canonical evidence object** — it is the verbatim provenance Klavity already prizes, and it is what lets a Sim reaction anchor to a replayable frame.

**Do NOT adopt** TestCafe's proxy/URL-rewrite transport (breaks on CSP/SPA/auth), WebDriver-as-primary (Nightwatch/Selenium overhead, weak auto-wait), or Cypress's in-browser cage (single-tab, same-origin, no Safari). **Do** keep a WebDriver-BiDi adapter as an *optional* path for mobile/native breadth later — the industry is converging on BiDi as the post-CDP standard.

**Steal the observation stack from the field's consensus:** accessibility-tree snapshot as the cheap **primary** text observation (Playwright `aria` snapshots, ref-mapped like `ref="abc123"`), Set-of-Marks vision as **fallback** for canvas/div-soup/visually-dense pages, and a DOM channel to recover `id/class/data-testid` attributes the a11y tree discards. Build the **ref→DOM-node mapping layer** (Browser-Use indices, Stagehand refs) regardless — it kills coordinate hallucination. Budget hard: Playwright MCP refs *everything* and bloats to 16–19k tokens/page; ref only *interactive* elements + region focus.

## 2. How personas-with-intent drive the browser

**Adopt the Planner/Executor split (Magnitude, Agent-E, Skyvern's Planner/Actor/Validator) — but make the Planner a Sim.** The persona's testing intent (grounded pain/want/love quotes) = the Planner goal. A cheap vision/grounding model (Qwen3-VL, Moondream-class) = the Executor doing per-step pixel/AOM grounding. This maps 1:1 onto your existing model-mix and is the single most direct blueprint in the corpus.

**Use a reactive "what's next from the current screen" loop, not rigid upfront plans.** The HN/Skyvern critique is consistent: pre-planned multi-step plans fail because the model guesses UI from training data. Persona exploration suits reactivity anyway.

**Assert on persona-GOAL completion, not element presence.** This is the load-bearing differentiator. testRigor/Mabl/Testim/Applitools all heal *element discovery* but are **intent-blind** — the canonical failure is "Submit Order heals to Save Draft, test green, order never placed." A Sim that knows "I'm trying to check out" can refuse a heal that doesn't satisfy the want, and treat the deviation as a *fileable bug*. Skyvern self-heals to *finish*; Klavity self-heals to *keep testing AND file the grounded bug* — the Validator step is a free bug detector.

**Extend persona traits with a testing-intent dimension.** PersonaTester's mindset/strategy/habit decomposition layered onto pain/want/love gives Sims a *how-to-test* axis, not just an emotional one.

## 3. Self-healing architecture (and the pitfalls)

**The pattern the whole field converged on — "intent-cache-heal":** cache the resolved selector keyed by `SHA256(method, normalized URL, DOM-hash, scope)` in SQLite/Turso; on replay do **passive snapshot-fingerprint validation**; on hit, replay deterministically with **zero LLM cost**; on miss (UI drift), invoke the vision-LLM to re-resolve and refresh the cache. This is Stagehand's published design, Skyvern's deterministic-cache-with-LLM-fallback, and workflow-use's 3-tier ladder (deterministic replay → semantic locator fallback → agent re-derivation that *rewrites the stored artifact*). **LLM tokens spent once at authoring, deterministic replay forever after** is the cost-control playbook that makes you cheaper than QA Wolf ($90K median, linear human cost) and the autonomous cohort.

**Candidate ranking = role + accessible-name → label → text → testid → structural → visual crop**, the settled Playwright/Testing-Library order. Store a **multi-signal fingerprint per element** (DOM attrs + accessible-name + visible text + screenshot crop + bbox) — Functionize's 5D embedding and Mabl's 35-attribute fingerprint validate this; you go further by re-resolving *visually/semantically* via the vision-LLM, surviving the **complete DOM rewrites** that collapse LCS/attribute methods (Healenium, Mabl, Testim).

**Non-negotiable guardrails (every source screams these):**
- **NEVER heal silently.** Every heal is a reviewable diff with a **confidence score + verbatim rationale** ("matched by role+label, 0.83"). Octomind's `octomind pull` reviewed-refactor model and Playwright Healer's edit-time-diff model are correct; runtime silent substitution is the trap.
- **High confidence gate, fail loud below it.** Healenium ships a *0.5* default (dangerous); use ≥0.9 and route low-confidence to a **distinct AMBER state**, not green. A low-confidence heal should **file a bug**, not pass — turning healing uncertainty into Klavity's native output.
- **Heal must NEVER override a failed assertion.** Functionize's hard rule; persona-grounded assertions stay immutable to healing. Otherwise you reproduce the enterprise-study failure mode: agents weakening `toBe(5)→toBeTruthy()` and silently deleting tests to fake green.
- **Diagnosis-FIRST.** Classify the failure (selector-drift / timing / test-data / runtime-error / visual / interaction-change) *before* remediating. Selector healing addresses only ~28% of real failures; ~45% is async-wait, and **bumping sleeps does nothing** (Microsoft finding). Klavity's vision-LLM "wait until the screenshot shows X" is itself a robust async-wait primitive — lean on it.

## 4. Combining static tests + adaptive persona exploration

**Hybrid is mandatory, not a compromise** — every serious practitioner says keep a deterministic Playwright path for happy paths, reserve AI for drift/novelty. Concretely:
- **Crystallize every passing persona run into ownable, exportable Playwright/Bun code** (Autify, Octomind, LaVague pattern). A flaky vision-LLM run is never the only record. This satisfies the "static tests too" requirement *and* counters the lock-in/black-box gripe leveled at testRigor/Mabl/Testsigma.
- **Two run modes:** deterministic regression (every commit, cached replay, fast/free/CI-gateable) vs. probabilistic exploratory persona runs (nightly, statistical pass/fail over N runs, never binary). The WebArena reality (~10–47% single-pass agent success; Operator ~61% honest ceiling) demands you **narrow the domain** — you test the *user's own app* with persona context + accumulated app knowledge, which beats general-agent numbers, and you average runs to tame non-determinism.
- **Support a mocked/hermetic mode AND real-backend mode.** Meticulous's deterministic-replay moat (virtualize time/timers/RAF, mock network from recordings, stub perf/random) is achievable on Bun by injecting a deterministic clock + fetch mock — a far cheaper path to "no flakiness" than retries. rrweb (MIT) is your recording layer for the MV3 extension; it gives fidelity, *not* determinism, so you must add the virtual-clock/network shim on top.

## 5. Learning from app changes

**Treat a needed re-plan as a "the product changed here" signal, not breakage.** This is the whitespace: *none* of the surveyed tools (frontier CU agents included) do persistent, persona-driven learning-from-change. When a Sim re-grounds, **persist the healed locator + the change-event + provenance** in SQLite, and **update the persona's trait/locator memory** so the next run is deterministic again. Mabl's adaptive-auto-heal (log heal outcomes + human approve/reject → confidence thresholds, 20–30%→70–80% autonomous over months) is the precedent — but you do it grounded in persona intent and explainable provenance, side-stepping Mabl's cold-start and heuristic-intent weakness (your vision-LLM judges on run #1; Mabl needs 10–12 passing runs for its visual model).

**Reuse the dedup/recurrence machinery you already shipped.** The governance/test-explosion problem ("which of 100 generated tests belong in CI?") is the real unsolved bottleneck per the Playwright-agents consensus. Klavity's `groundQuote` + verified tri-state + recurrence-bump (built for bug reports) is *directly transferable* to (a) deciding which generated tests/regressions matter and (b) the **flake-vs-real-bug disambiguator** — replay a suspected regression on a known-good baseline (Dropbox Athena's confirm-by-replay) before filing, and cluster-dedup root causes so one networking fix deflakes a whole cluster.

## 6. Klavity's unique, defensible wedge

**You already own the two hardest pieces every incumbent bolts on:**
1. **A cheap vision-LLM react pipeline** = the *perceptual oracle* that catches "visually wrong but DOM-valid" bugs (10.50 vs 10.51, layout breakage, button-rendered-but-unreachable) that AOM-only Playwright MCP and DOM-only healers miss — *and* on run #1 with zero ML cold-start (vs Mabl/Applitools baseline training).
2. **Provenance-tracked personas (pain/want/love grounded in verbatim quotes)** = the *correctness/intent oracle* the Playwright Healer, Stagehand, Functionize, and Skyvern all explicitly lack. This is the missing "business-intent spec" the enterprise autonomous-repair study said is essential, and the answer to record-and-replay's intent-inference gap ("was the hover intentional?").

**Critical risk to manage — you ARE an AI bug-reporter, the slop category.** WebProber measured **85% false positives** filing VLM bugs in the wild; curl rejects ~66% of AI vuln reports as junk. --- TOP TAKEAWAYS ---
- Don't compete on self-healing locators — that lane is commoditized (Octomind, a polished agentic-Playwright product, was discontinued May 2026 once Playwright shipped free native agents). Compete on the persona INTENT ORACLE + vision perceptual oracle you already own — the exact two things Playwright Healer, Stagehand, Functionize, and Skyvern all explicitly lack.
- Architecture: Playwright/CDP deterministic floor (on Bun, trace as evidence object) + Planner/Executor split where the SIM is the Planner (Magnitude/Agent-E blueprint) + 'intent-cache-heal' in SQLite (Stagehand pattern: cache resolved selector, passive-validate, zero-LLM replay, vision-LLM only on drift). LLM tokens spent once at authoring, deterministic replay forever — the cost playbook that undercuts QA Wolf's linear human cost.
- Self-healing guardrails are non-negotiable and the whole corpus screams them: NEVER heal silently (reviewable diff + confidence + rationale), high gate (≥0.9, AMBER-not-green below it, low-confidence FILES a bug), heal must NEVER override a failed assertion (Functionize rule), and DIAGNOSE failure class first (selector healing fixes only ~28%; ~45% is async-wait where bumping sleeps does nothing). Silent heal-to-wrong-element (Submit→Save Draft, green-but-broken) is the canonical false-green that kills suite trust.
- Assert on persona-GOAL completion, not element presence — a Sim that knows 'I'm trying to check out' refuses a wrong heal and treats the deviation as a grounded fileable bug (Validator step = free bug detector). This closes the silent-false-pass gap that every attribute/pixel self-healer (testRigor 87%, Mabl 91%) suffers.
- Hybrid by design: crystallize every passing persona run into exportable Playwright/Bun code (ownable, no lock-in), run deterministic regression on every commit vs statistical exploratory persona runs nightly, and add a Meticulous-style deterministic-replay shim (virtual clock + network mock on rrweb capture) for a real 'no-flakiness' claim. Narrow domain (the user's own app + persona context) to beat the ~60% general-agent reliability ceiling.
- Learn from change as whitespace nobody fills: persist healed locator + change-event + provenance in SQLite and update persona trait/locator memory so next run is deterministic again; reuse the already-shipped groundQuote + dedup + recurrence machinery for flake-vs-real-bug disambiguation (confirm-by-replay on known-good baseline) and test-governance. Manage the existential slop risk (VLM bug-filing = 85% false positives in the wild) — grounding + reproducible evidence + confidence gate is the moat, and publish a precision metric.
